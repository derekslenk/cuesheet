# Phase 3B (raw) — Documentation Review

**Verdict:** Documentation is unusually rich for a solo tool — the freshest subsystem (HTML overlay) is exemplary and accurate against code — but carries three event-day-grade operator gaps (boot order, env-var/auth reference, stale OBS setup) and multiple accuracy bugs where docs contradict shipped code (version badges, API envelope spec, OBS source defaults).

## CRITICAL — operator-facing, can cause event-day failure
- **C1. Webui-before-supervisor boot order undocumented in every operator runbook.** Missing from RUNBOOK_EVENT.md, OBS_SETUP.md, RUNBOOK_FALLBACK.md, README quick-start. Documented only in scripts/streamlink-supervisor/README.md:90-95 (operator won't read at boot). Code-confirmed: compiled supervisor (index.bun.ts→bunDatabase.ts→streamSpecsLoader.ts) only SELECTs, no CREATE TABLE; schema creation solely in webui lib/database.ts:50-100. Cold start with supervisor first → `no such table`/SQLITE_CANTOPEN. tsx dev entry imports lib/database and masks this in dev → production-binary-only surprise. Fix: add "Cold start ordering" to RUNBOOK_EVENT T-minus + README quick-start, with exact failure symptom.
- **C2. No single env-var reference in event runbook; security posture undocumented.** RUNBOOK_EVENT.md grep for API_KEY|EVENT_KEY|LABEL_RENDERER|OBS_BULK_PACE_MS|TWITCH_CLIENT|STREAMS_TABLE|loopback|firewall|bind → ZERO matches. Env vars scattered, no canonical table. Security (S-F1): middleware fail-open (16-19) + bypasses auth for Host localhost/127/192.168 (23) even when API_KEY set. README/API.md describe bypass mechanically but never state intended deployment posture (bind loopback/firewall — anyone on 192.168 trusted). Fix: (1) single "Environment reference" table (every knob + default + which process reads it). (2) "Security posture" note: trusted LAN only, bind loopback or firewall, NOT safe on untrusted network. NOTE: OBS_BULK_PACE_MS only found read in obsClient.js:26-30 — confirm still a live knob before documenting.
- **C3. docs/OBS_SETUP.md:57-62 stale — contradicts shipped media-source pipeline.** Says "Add Browser Sources... Width 1600 Height 900" (legacy). Default now (STREAM_USE_FFMPEG unset/true, README:365, lib/streamLabel.js) creates ffmpeg_source Media Source fed by UDP relay + separate transparent HTML browser source label at /overlay/stream/<id>. Setup guide describes neither. Fix: update section 4 for Media Source pipeline + STREAM_USE_FFMPEG=false fallback + per-stream HTML label; cross-link stream-labels-runbook.md.

## HIGH — accuracy bugs + missing API contract
- **H1. No canonical API response-envelope spec; docs/API.md documents one shape, code ships five.** API.md:20-37 documents single {success,data,message}. Audit of all 25 routes found 5 shapes: A true apiHelpers {success,data,timestamp} (only 5/25); A' hand-rolled {success:...} ad-hoc keys no timestamp (10); B overlay {ok} (intentional); C bare/ad-hoc {message}/{error}/raw row/{viewers} (obsStatus, setActive, addStream, streams/[id], teams/[teamId], supervisor/health); D non-JSON new Response (preview binary HLS). Concrete contradictions: getCurrentScene doc says data.currentScene, route returns data.sceneName; obsStatus doc says wrapped, route returns bare; documented error {success:false,error,message} matches neither apiHelpers ({error,message,details,timestamp} no success) nor most routes; overlay/[id]/viewers mixes shapes (success {viewers} bare, error {ok:false,id}). Fix: document actual per-route contract OR state overlay+preview deliberate exceptions + rest migrating to apiHelpers; at minimum fix the 3 contradictions.
- **H2. 10/25 routes absent from API.md.** Missing: all 3 overlay routes, all 4 supervisor proxy routes, counts, obsPlaybackSettings, preview/[...slug]. Supervisor control + overlay health are operator-facing event-day tools. README:466-474 claims API.md is "Complete REST API reference" — ~60% complete. Fix: add them or drop "Complete".
- **H3. Stale framework versions.** README:8 badge "Next.js 15"; README:456 "Next.js 15 with React 19"; AGENTS.md:6 "Next.js 15". package.json ships next ^16.2.9, react ^19.2.7, typescript ^6.0.3, sqlite3 ^6.0.1, node>=22, commander ^15 (PR #26-29). Fix: bump to Next 16/React 19/TS 6/Node 22.
- **H4. No changelog/migration guide.** Glob *CHANGELOG*/*MIGRATION*/*UPGRADE* → zero. Three shipped migrations unrecorded: Next 15→16/React 19/sqlite3 6/Node-20-dropped (only in git history + private memory); OBS-native→HTML labels (adequately covered in stream-labels-runbook.md:133-138, not a gap); browser→ffmpeg media source (only one-line script entry, no when/why/rollback). Fix: short docs/CHANGELOG.md capturing dep upgrade (+ rollback-kit pointer) + browser→media paragraph.
