# Phase 1B (raw) — Architecture & Structural Integrity Review

**Scope:** Whole repo (~27k LOC). Next.js 16 App Router + React 19 + TS 6 control plane. Three runtimes (Node/Next, Bun CLI+supervisor, tsx scripts), three tsconfigs. Priority on overlay/stream-label subsystem + Twitch.

**Overall:** Structurally sound, deliberately pragmatic single-host application, correctly architected as such. Shared-kernel partition (`lib/constants.ts`, `lib/relayPort.ts`) across runtimes is well-designed. The new overlay/Twitch subsystem is the best-architected part. Primary debt: HTTP API surface (inconsistent envelopes), dual DB driver/schema-ownership split, duplicated constants lacking sync mechanism. None are event-blockers.

## CRITICAL
- **C1. Schema ownership split across two DB drivers with no shared DDL source of truth.** `lib/database.ts:50-100` (Node/sqlite3 owns CREATE TABLE + ensureColumns) vs `scripts/streamlink-supervisor/bunDatabase.ts:15-29` (Bun/bun:sqlite, no schema logic) + `runtime.ts:121-144` (Bun supervisor writes `disabled`). Two processes, two drivers, one `sources.db`. Supervisor never creates/migrates tables — assumes webui booted first (undocumented, unenforced boot order). PRAGMA setups already drifted (Bun adds `wal_autocheckpoint=1000` that Node opener lacks). Fix: extract declarative schema constant consumed by both, or add a Bun-side PRAGMA table_info guard that fails fast; document webui-first invariant.

## HIGH
- **H1. Three competing API response contracts across 24 routes.** `lib/apiHelpers.ts` standard envelope used by only ~5 routes (streams, teams, counts, getActive, getTeamName). ~19 hand-roll: ad-hoc `{message,...}` (addStream:208, teams/[teamId]:78), overlay `{ok}` (overlay/[id]:70, overlay/health:38). Error contracts + status codes diverge (setActive 400 vs supervisor start 404 for same conceptual case). Fix: adopt apiHelpers envelope as default for control-plane routes; formally bless overlay `{ok}` as documented exception; codify in app/api/README.md or ADR; add a response-shape guard test (team already uses tableNameHardcodeGuard.test.ts pattern).
- **H2. `lib/obsClient.js` is a 1,404-line untyped god-module mixing 5 responsibilities.** Connection mgr + scene/group CRUD + switcher manipulator + label orchestrator (legacy+HTML) + teardown. Imported by 6+ routes. Being `.js`, sits outside TS; callers use local `interface OBSClient` shims (addStream/route.ts:9-11) that can drift. Newest label logic correctly extracted to streamLabel.js/labelLayout.js (because obs-websocket-js is ESM-only, can't require under jest). Continue that extraction: thin obsConnection + pure obsScenePlan (DI) + thin glue; migrate to .ts incrementally. Multi-session refactor.
- **H3. Default-color palette duplicated across runtimes/renderers, sync by comment only.** `lib/overlayData.ts:15-19` EVENT_DEFAULT_COLORS (#472f5a/#e0d9f1 hex) vs `lib/labelLayout.js:82-83` PLATE_COLOR=0xFF5A2F47/ACCENT_COLOR=0xFFF1D9E0 (ABGR). overlayData.ts:12-14 documents "Mirrors lib/labelLayout.js" — human discipline only, no shared constant, no test. ABGR↔hex byte-swap error-prone. Latent: two renderers can diverge. Fix: define palette once (hex), derive ABGR with hexToAbgr() helper; add test asserting equality. Also route inline `_stream` name construction (addStream:184-186, teams/[teamId]:125-127) through buildStreamGroupName.

## MEDIUM
- **M1. Multi-event isolation = lexical table-name suffixing in one shared DB, no per-event row scoping or cross-process EVENT_KEY check.** constants.ts:27-64 (EVENT_KEY→streams_<key>), database.ts:107 (single sources.db). Resolved at module-eval from process.env.EVENT_KEY; if webui and supervisor launch with different EVENT_KEY they silently use different tables, no error. Fix: supervisor /health includes resolved EVENT_KEY; webui compares + warns on mismatch.
- **M2. No authorization model; only LAN/localhost middleware bypass.** middleware.ts:1-41 skips auth if API_KEY unset (:16-19) and bypasses for Host starting localhost/127.0.0.1/192.168 (:22-25, spoofable Host header). Mutating routes as open as reads. Accepted posture for single-operator LAN tool; flagged for completeness. If surface broadens, add withAuth wrapper in apiHelpers; prefer interface binding over Host trust.
- **M3. Three DB-access idioms coexist, no convention.** `lib/db.ts` withDb wrapper, direct getDatabase() (overlay/[id]:48, teams/[teamId]:39, addStream:91), supervisor MinimalDb. addStream uses both withDb (:25,165) AND raw getDatabase() (:91,222) in same file. withDb is thin (no tx/error mapping). No repository layer — raw SQL with interpolated table names scattered (stream+team join in overlay/[id]:53-58, streams:9-16, addStream). Fix: commit to withDb (with real value) or remove it; extract lib/repositories/ (streamRepo, teamRepo).
- **M4. Overlay live-data: batched Twitch capability unused; per-label poll + two uncoordinated TTLs.** twitch.ts:139-198 server viewerCache 25s TTL batched, but route calls single-login getViewerCount (viewers/route.ts:52) one HTTP/label, each with client 30s poll (StreamLabel.tsx:66). N labels → N polls/invocations; batch API dead. 25s/30s magic numbers in different layers. Works (cache makes it correct). Post-event: batched /api/overlay/viewers?ids=... or drop batch path; co-locate constants.

## LOW
- **L1.** `lib/overlayData.ts` is exemplary (contract / row / pure resolvers / validators, no I/O, unit-testable, optional-column tolerant). Template for the repository extraction (M3). Positive finding.
- **L2.** `score` is a permanent-null placeholder field in shipped contract (overlayData.ts:35-44,137). Well-documented YAGNI; keep or drop.
- **L3.** Inconsistent module systems (CJS streamLabel/labelLayout/obsClient vs ESM rest) + import styles (@/lib alias vs ../../../lib relative). CJS is deliberate (jest/ESM). Standardize alias via no-restricted-imports lint; leave CJS until obsClient TS migration.
- **L4.** overlayMetrics.ts process-local counters invisible across multi-process boundary; operator consults two health panels (/api/overlay/health + supervisor dashboard). Optional unified view via supervisorClient.fetchSupervisorHealth.

## Summary
C1 schema ownership split (the one Critical, ordering hazard). H1 envelope sprawl, H2 obsClient monolith, H3 palette duplication = top post-event refactors. None event-blockers. Freshest work (overlay+Twitch) is highest-quality and the template for the rest.
