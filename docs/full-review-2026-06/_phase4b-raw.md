# Phase 4B (raw) — DevOps / CI-CD & Operational Review

**Calibration:** Solo-operator, single-host event tool. Bar: can the pipeline catch a broken Windows Bun binary or broken deploy before event day?

**Executive summary:** Well-run repo for its size. ci-ok aggregate-gate genuinely good (solves path-filter-deadlock + matrix-rename-staleness). Rollback kit + event-safe git tag real (event-safe-2026-06-13 present). Two detailed scenario-indexed runbooks. Supervisor binary boot-smoked in CI. Secret hygiene clean — no .env tracked, live Twitch token NEVER in history. **Headline risk: parity gap in the most event-critical gate** — supervisor smoke builds+boots the LINUX binary on ubuntu-latest, but the host ships the WINDOWS cross-compile. The bug class the smoke exists to catch is exactly the class that can differ Linux↔Windows, and A-C1 boot-order only manifests in the compiled path. Secondary: no supply-chain scanning in CI; actions pinned to mutable tags; manual deploy with tribal boot-order trap; unfilled TODO(operator) in emergency runbook.

## CRITICAL
- **C-1. Blocking supervisor smoke validates the WRONG OS binary.** build.yml job supervisor-binary runs ubuntu-latest → npm run supervisor:smoke → smokeSupervisorBinary.mjs:50 runs supervisor:build (host-native = Linux ELF). Deployed artifact is dist/supervisor.exe (supervisor:build:win --target=bun-windows-x64, NSSM points at it). **Windows supervisor binary NEVER built/booted in CI.** bun build --compile strips types w/o checking; cross-compiled targets diverge at runtime (paths, bun:sqlite, signals). A-C1 boot-order failure manifests ONLY in compiled path — and smoke seeds its own throwaway DB so never reproduces A-C1 even on right OS. False-green gate reaches event day. Fix: (1) add Windows leg (windows-latest, setup-bun@v2 supports it); (2) build the SAME target that ships (supervisor:build:win on Windows); (3) add a smoke assertion reproducing A-C1: boot against FILE_DIRECTORY with NO sources.db/season table, assert fast+loud non-zero exit (boot-order gate-enforced not just documented).

## HIGH
- **H-1. Boot-order dependency (A-C1) enforced nowhere in deploy automation.** Supervisor README explicit: assumes webui created schema first, else SQLITE_CANTOPEN/no such table. doctor.ts:113 explicitly does NOT check sources.db/season table ("directory need not exist yet"). NSSM auto-restarts (AppRestartDelay 1000) → on cold host where webui hasn't seeded, supervisor enters 1s NSSM restart-crash loop. RUNBOOK_EVENT §2 doesn't sequence webui-before-supervisor for fresh DB. Fix: DB/schema preflight in doctor/supervisor startup (print actionable line + clean exit, don't let NSSM hammer); raise NSSM AppThrottle; explicit ordered runbook step.
- **H-2. No supply-chain or secret scanning in CI.** No npm audit, dependency-review, CodeQL, gitleaks/trufflehog in .github or .forgejo. Renovate (renovate.json5) configured well but gated on installing Mend app (NOT yet installed). Zero automated dep-vuln/secret-leak signal today. .env.local holds live Twitch creds + middleware fail-open → accidental secret commit uncaught. Fix: add security job (npm audit --audit-level=high advisory→blocking via ci-ok; dependency-review-action on PRs; gitleaks on PRs); install Renovate post-event.

## MEDIUM
- **M-1. CI actions pinned to mutable tags not SHAs** (checkout@v4, setup-node@v4, upload-artifact@v4, setup-bun@v2, paths-filter@v3, action-gh-release@v2, claude-code-action@v1). release.yml has contents:write, Claude workflows id-token:write — high blast radii. Fix: pin third-party (dorny/oven-sh/softprops) to full SHAs; Renovate github-actions manager keeps updated. First-party actions/* can stay on tags.
- **M-2. .forgejo mirror CI drifted, materially weaker.** Runs only type-check/lint/sqlite-audit/build. OMITS npm test, type-check:deck, type-check:bun, supervisor smoke, CLI binary build/smoke. Uses upload-artifact@v3 (EOL; GitHub on v4). AGENTS.md says keep aligned — they aren't. If Forgejo ever source of truth (S7 host-inaccessible runbook), passes builds GitHub rejects. Fix: bring to parity OR comment-demote to "advisory only — not a release gate"; bump v3→v4.
- **M-3. No job timeout-minutes, no concurrency cancellation.** Hung Bun build/smoke can run to GitHub 6-hour default; rapid pushes stack redundant runs; stuck ci-ok blocks merges silently. Fix: timeout-minutes per job (web ~20, binary/smoke ~15) + concurrency:{group:ci-${{github.ref}},cancel-in-progress:true}.
- **M-4. Runbooks ship with unfilled TODO(operator) in emergency paths.** RUNBOOK_FALLBACK.md: entire §3 hotkey table (BRB, Starting, 4/2/1-Screen, mute — all TODO), §5 plugin-free manual source-select, §7 idle-Streamlink naming, §8 USE_FFMPEG_SOURCE flag name, §10 webui restart cmd. BRB is the single most-invoked safety action and its binding is a TODO. Fix: pre-event dry-run pass (runbook has §11 dry-run checklist), replace every TODO; treat "zero TODOs in fallback runbook" as a doors-don't-open gate.
- **M-5. Stale committed checksums + stale release/ tree.** release/ gitignored (good) but working tree has release/SHA256SUMS.txt + 4 binaries dated 2026-06-08 while package 0.1.0; root has 7 multi-hundred-MB *.bun-build litter (~700MB). release.yml regenerates SHA per tag so committed-looking copy is local-staleness trap. Fix: git clean -fdX periodically; extend "verify deployed version via /health" into RUNBOOK_EVENT §2.

## LOW
- **L-1. Coverage gate (70%) intentionally not enforced — RIGHT CALL, make visible.** CI runs npm test not test:ci; global ~32%; comment documents why. No change. Optional: non-blocking coverage trend artifact. Don't raise threshold pre-event.
- **L-2. ci-ok needs: coverage good but verify deck/bun type-checks stay wired.** ci-ok aggregates [changes, web, supervisor-binary, cli-binary]. web runs type-check + type-check:deck + lint + npm test + audit:sqlite-opens + build; supervisor-binary runs type-check:bun + smoke; cli-binary builds+smokes per-OS. All three tsconfigs + lint + sqlite audit + both smokes ARE in the gate — well-constructed. Fragility: deck/bun type-checks wired inside unrelated jobs to avoid ruleset edit — non-obvious, a future job split could silently drop a type gate. Keep comments; consider package.json comment noting which CI job owns each type-check:*.
- **L-3. Per-job permissions hardening incomplete.** build.yml sets permissions only on changes job; web/supervisor-binary/cli-binary/ci-ok inherit repo-default. Fix: top-level permissions:{contents:read}, jobs opt up explicitly.
- **L-4. Health/control endpoints unauthenticated (loopback-only) — acceptable, coupled to S-F1.** Supervisor /health + /streams/* control (MUTATE db: flip disabled, start/stop/restart) bind 127.0.0.1 only no auth by design (right model). Webui middleware S-F1 (fail-open + spoofable Host 192.168.* bypass) is the soft spot on a 0.0.0.0-bound Next. Fix: bind Next to loopback or firewall :3000 on host + document in runbook.
